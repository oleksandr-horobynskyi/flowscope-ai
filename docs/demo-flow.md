# Demo Flow

## Entry Point

The user lands on the FlowScope AI landing page and sees the core promise: describe a business problem and receive a structured action path.

## Step-by-Step Demo

1. The user clicks `Launch in Telegram`.
2. The Telegram bot asks the user to describe the business problem in plain language.
3. The user submits a hiring-related problem, for example:
   - "We do not have an HR department."
   - "We received many CVs and reviewing them manually takes too long."
4. FlowScope AI analyzes the request and identifies it as a recruitment case.
5. The assistant routes the user into the HR screening branch.
6. The bot asks for a vacancy description or the core candidate requirements.
7. The workflow prepares a structured output based on the submitted hiring request.
8. The result is returned as a practical next step for the user.

## What the User Receives

- structured interpretation of the problem
- recognized workflow type
- a recommended scenario
- a recruitment-focused output in the MVP

## Demo Message

The demo is designed to prove one central point:

FlowScope AI helps a small business move from a messy request to a structured action path without needing to understand which AI tool should be used first.
